Introduction

Hello there!


I am Obsidian, and I have been working with modifying phpBB for about two years now. I'm writing this to help get you started with learning to build MODs for phpBB 3.0.x (codenamed "Olympus"), and to provide you with the resources and tools you need to succeed.

Not sure what a phpBB MOD is?


Well, I'll explain.
Because of phpBB being an open-source project, you are free to modify it to suit your needs as you wish. When you modify your copy of phpBB, document the changes you have made, and distribute those changes, in essence you have released a MOD.
In reality, it is a bit more involved than that, but don't sweat it; we'll get to that later.

What all is in this welcome package?


This welcome package contains the guide that you are reading now, and several tools that are used by many MOD authors that you may want to use yourself. A more in-depth overview of the tools included will appear later on in this guide.

What will this guide cover?


Actually, it will be a broad range of things, including:

  • The development process

  • Tools of the trade

  • Coding and development resources

  • Utilizing built-in phpBB3 functionality

  • Humorous source code comments in phpBB3

MODing Benefits


Obsidian explained what a MOD is, why do them at all? Aside from gaining some sort of new functionality for your board, you enable others to have the same feature; not to mention, you are contributing to the Open Source community.

Explore phpBB


Before you begin to read though MOD documentation, I highly recommend doing one thing: Explore phpBB. Not just the web interface, but the code that powers it. Pick some feature and find the code that makes it tick. You don't have to understand it right away, but it is easier to learn when you have questions about the code to be answered.

Coding for phpBB: What you will need

So, what do I need to become a phpBB MOD Author?


Not much, actually. Here's the basics of what you will need.

A decent editor


I'm sorry, but Windows Notepad and Mac's TextEdit just will not do. We'll cover some free third-party editors later on.

Basic PHP knowledge


If you are altering phpBB, you probably want to know what you are doing. You don't need to know everything about PHP, but being comfortable with if(), foreach(), and variables will be a great help. If you don't know, Google is your friend for finding some tutorials on coding in PHP.

XHTML and CSS experience


XHTML 1.0 and CSS are the languages of choice for formatting output to a user's web browser with phpBB. You'll need to know these two so that the user can actually see what your MOD can do...and so that you can make your MODs look pretty.

A way to test your MOD


You're going to need a phpBB installation to test your MOD with. A localhost test server bundle (such as Apache, PHP, and MySQL, or a prepackaged compilation of the three) or an online site will do fine. We'll cover some things you can use for a localhost server later on, as well.

AutoMOD



AutoMOD is a tool designed to parse and automatically install MODX MODifications for phpBB3.
It also has the ability to uninstall MODs.
It is a useful tool for MOD authors and end users alike: All MODs are installed with AutoMOD prior to being accepted in the MODs Database. As such, any MOD listed in the MOD Database should install with AutoMOD.

Included version: 1.0.0 RC 3

Available at: http://www.phpbb.com/mods/automod/

Related Links:
MODX - phpBB.com MODs Database - phpBB.com MOD Tools Bug Tracker - AutoMOD SVN Repository


Warning!
AutoMOD is development quality code, and as such it is recommended that backups be taken whenever using AutoMOD in case of data loss or corruption, or damage caused by a third-party modification.

UMIL



UMIL (Unified MOD Install Library) is a utility that enables MOD Authors to create simple installation scripts for their MOD that allow the user to install, update, or uninstall a Modification quickly.
Released under the GNU GPL license, UMIL is included within MOD Packages to make MOD Installations simple and automatic.

UMIL performs basic database actions that would be required to install a MOD; UMIL does not perform file edits.
AutoMOD is the tool of choice for file edits and installing MODs from a MODX file. AutoMOD does not perform module, permission installations, etc.; this is the purpose for which UMIL was developed.

What can UMIL do?

The Unified MOD Install Library can perform any of the following operations as specified by a UMIF (Unified MOD Install File) provided by the Modification:

  • Add, update, or remove configs

  • Add or remove modules

  • Add or remove permissions

  • Set or unset permission options from roles and groups

  • Add or remove database tables

  • Add, update, or remove table columns

  • Add or remove table keys/indexes

  • Purge / Refresh Caches



Included version: 1.0.1-pl1

Available at: http://www.phpbb.com/mods/umil/

Related links: UMIL documentation on the phpBB Wiki - UMIF Creation Tool - phpBB.com MOD Tools Bug Tracker - UMIL SVN Repository

MODX Quickstart Files



MODX is a format by the phpBB MOD Team that is used to describe, in detail, changes between web applications suitable for distribution.
It is an XML-based format for both MOD writers and users alike.

Included version: 1.2.5

Available at: http://www.phpbb.com/mods/modx/
